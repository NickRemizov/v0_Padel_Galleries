# Galeries - Padel Tournament Photo Management

Полнофункциональная система управления фотографиями с турниров по паделу с автоматическим распознаванием лиц.

## 🎯 Основные возможности

### Управление контентом
- ✅ **Галереи событий** - создание, редактирование, удаление галерей
- ✅ **Загрузка фото** - drag & drop, пакетная загрузка
- ✅ **Управление людьми** - игроки с профилями, рейтингами, соцсетями
- ✅ **Организаторы, локации, фотографы** - полное управление метаданными

### Распознавание лиц
- ✅ **Автоматическое распознавание** - InsightFace + HNSWLIB
- ✅ **Кластеризация неизвестных лиц** - HDBSCAN группировка
- ✅ **Ручная верификация** - точное назначение людей на фото
- ✅ **Настройки качества** - фильтры по размеру, резкости, уверенности
- ✅ **Пакетная обработка** - массовое подтверждение и удаление

### Интеграции
- ✅ **Telegram бот** - отправка фото пользователям
- ✅ **Vercel Blob Storage** - хранение изображений
- ✅ **Supabase** - PostgreSQL база данных
- ✅ **Google OAuth** - аутентификация администраторов

### Проверка целостности
- ✅ **15 проверок БД** - автоматическая диагностика проблем
- ✅ **7 автоисправлений** - восстановление согласованности данных
- ✅ **Детальные отчеты** - просмотр проблемных записей

## 🚀 Быстрый старт

### Предварительные требования

- Node.js 18+ и npm
- Python 3.9+
- Supabase аккаунт
- Vercel аккаунт (для Blob Storage)

### Локальная разработка

1. **Клонируйте репозиторий:**
\`\`\`bash
git clone <repository-url>
cd galeries
\`\`\`

2. **Установите зависимости Next.js:**
\`\`\`bash
npm install
\`\`\`

3. **Установите зависимости Python:**
\`\`\`bash
cd python
pip install -r requirements.txt
\`\`\`

4. **Настройте переменные окружения:**

Создайте `.env.local` в корне проекта:
\`\`\`env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# Vercel Blob
BLOB_READ_WRITE_TOKEN=your_blob_token

# FastAPI Backend
FASTAPI_URL=http://localhost:8001
NEXT_PUBLIC_FASTAPI_URL=http://localhost:8001

# Telegram Bot (опционально)
TELEGRAM_BOT_TOKEN=your_bot_token
NEXT_PUBLIC_TELEGRAM_BOT_USERNAME=your_bot_username
\`\`\`

Создайте `.env` в папке `python/`:
\`\`\`env
SUPABASE_URL=your_supabase_url
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
JWT_SECRET_KEY=your_jwt_secret
ALLOWED_ORIGINS=http://localhost:3000
\`\`\`

5. **Запустите оба сервера:**

Terminal 1 (Next.js):
\`\`\`bash
npm run dev
\`\`\`

Terminal 2 (FastAPI):
\`\`\`bash
cd python
python main.py
\`\`\`

6. **Откройте в браузере:**
   - Next.js: http://localhost:3000
   - FastAPI: http://localhost:8001
   - API Docs: http://localhost:8001/docs

## 📊 Статусы фото

Система использует визуальные бейджи для отображения статуса обработки фото:

- **Без бейджа** - Новое фото, не проходило распознавание
- **NFD** (серый) - No Faces Detected, лица не обнаружены
- **XX/YY** (оранжевый) - XX неопознанных лиц из YY всего
- **XX%** (синий) - Минимальная уверенность распознавания
- **✓** (зеленый) - Все лица верифицированы

## 🔐 Аутентификация

Система использует Supabase Auth с Google OAuth:

1. Администраторы входят через Google аккаунт
2. Доступ к админ-панели только для авторизованных пользователей
3. Публичные галереи доступны всем без авторизации

## 🐛 Известные ограничения

1. **Контекстное распознавание** - настройка есть, но логика не реализована
2. **Категории людей** - ENUM типы не созданы в БД
3. **Единичные фото в кластерах** - не показываются (HDBSCAN помечает как шум)
4. **Детальная информация о распознавании** - UI готов, но данные не приходят с бэкенда

См. [CURRENT_STATUS.md](CURRENT_STATUS.md) для полного списка.
